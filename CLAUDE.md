# Семейное древо

Русскоязычный сайт семейного древа. Просмотр открыт всем; редактирование — за паролем. Хостинг — GitHub Pages (билд `dist/`). Данные — base64-JSON в репозитории.

## Стек

- React 18 + Vite 5 (`@vitejs/plugin-react`)
- `@xyflow/react` (React Flow v12) — рендер карточек-нод и связей
- Раскладка — собственная (Sugiyama-lite), без внешних библиотек дерева

## Команды

```bash
npm run dev      # дев-сервер Vite (localhost:5173)
npm test         # node --test test/ (чистые модули, без браузера)
npm run build    # vite build → dist/
npm run preview  # предпросмотр сборки
node scripts/seed.mjs   # пересоздать public/data.json из каркаса
```

## Архитектура

- **Данные** (`src/data.js`): `encodeData`/`decodeData` (base64 JSON), `buildGraph` →
  `Map<id, person>`. Персона: `{ id, fio, sex, status, spouses[], children[], parents[], ... }`.
  `parents` достраивается из `children` обоих родителей. Имена НЕ уникальны — id уникальны,
  по имени людей не сливать (повторяющиеся имена = разные люди).
- **Раскладка** (`src/layout-core.js`, чистый JS, тестируется в node):
  - `buildLayout(graph)` → `{ nodes, edges }` для React Flow.
  - **Sugiyama-lite**: слой = поколение (`computeGenerations`, длиннейший путь от
    основателя; супруги выравниваются в один слой через max). Порядок внутри слоя —
    DFS-preorder (`dfsOrder`, семьи кучкуются, супруг сразу за партнёром). Узлы пакуются
    по слотам слева-направо → карточки не налезают. Пара муж/жена = один юнит = соседние
    слоты. Длинные линии между поколениями/родами допустимы.
  - `buildEdges`: родитель→ребёнок (стрелка, `smoothstep`), супруги (пунктир + ♥, без стрелки).
  - `computeBloodFamily`: патрилинейный корень рода для каждого; `familyColors` → цвет-акцент
    карточки («кровь рода»).
  - `computeClusters`/`connectedComponents` — вспомогательные (кластеры по патрилинейному
    роду, связные компоненты). Сейчас раскладка их не использует, но они в тестах.
- **`src/layout.js`** — точка импорта для браузера, просто `export *` из `layout-core.js`.
- **Карточка** (`src/card.js`): `renderCardHTML(person)` — HTML карточки. Переиспользуется
  внутри `PersonNode.jsx` через `dangerouslySetInnerHTML` (стиль + тесты карточки целы).
  Цвет рода передаётся CSS-переменной `--blood` на обёртке ноды, НЕ внутри `renderCardHTML`.
- **UI** (`src/App.jsx`): `ReactFlowProvider`, загрузка `data.json` через
  `import.meta.env.BASE_URL`, drag нод, режим калькулятора родства (клик 2 карточки →
  `findRelation`), модалки. `PersonModal.jsx` (просмотр/правка/добавить родственника),
  `KinshipModal.jsx`.
- **Родство** (`src/kinship.js`, `src/terms.js`): кровное + N-юродные + свойство (по браку).
- **Прочее**: `auth.js` (пароль), `photo.js` (ресайз фото), `save.js` (коммит в GitHub).

## Конвенции

- Карточка: ФИО, БИО, место рождения, фото (силуэт ♂/♀ если нет), статус (жив/упокоен/
  неизвестно), национальность, контакты (живым), место упокоения как ссылка Яндекс.Карт (усопшим).
- В `scripts/seed.mjs` оба супруга дублируют `children`, чтобы `buildGraph` проставил обоих родителями.
- Фото — `public/photos/`, данные — `public/data.json` (Vite копирует в `dist/`).
- `vite.config.js`: `base: './'` — относительные пути для подпапки GitHub Pages.

## Тесты

`test/` — чистые модули (node `--test`), браузерных зависимостей нет. `layout-core.js`
тестируется напрямую. При правках раскладки проверять: все персоны отрисованы, нет
наложений (уникальные позиции), супруги на соседних слотах в одной строке.

## Деплой (не сделано)

GitHub Pages через Actions (билд `dist/` → Pages). Перед деплоем заполнить `config.js`:
реальный `repo` (owner/repo) и `passwordB64`. Репа публичная — пароль слабая защита (принято).
Владелец проверяет локально перед деплоем.
