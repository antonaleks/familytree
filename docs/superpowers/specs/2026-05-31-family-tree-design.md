# Семейное древо — дизайн

Дата: 2026-05-31

## Цель

Сайт-генеалогическое древо: смотреть и заполнять краткую информацию о родственниках.
По умолчанию открывается в режиме чтения. Редактирование — после ввода пароля.
Хостинг: GitHub Pages (статика, публичная репа).

## Стек и архитектура

- Без сборщика. Чистый HTML/CSS/JS (ES modules).
- Рендер дерева: **family-chart** (donatso, D3-based, MIT). Кастомный рендер карточек.
- Калькулятор родства: свой модуль поверх JSON-графа.
- Данные: `data.json` (base64-кодирован), фото в `/photos/`.

Модули (каждый — одна задача, тестируется отдельно):
- `data.js` — загрузка/декод base64 JSON, нормализация в граф персон.
- `card.js` — рендер бордо-карточки (просмотр + режим правки).
- `tree.js` — инициализация family-chart, вертикальная раскладка, сердечки браков, «+».
- `auth.js` — проверка пароля (base64), переключение режима админа.
- `save.js` — коммит data.json + фото через GitHub API по токену.
- `photo.js` — загрузка фото, ресайз до ~400px в браузере.
- `kinship.js` — выбор 2 персон → термин родства + пояснение.

## Модель данных

Персона:
```
{
  "id": "string",
  "fio": "Фамилия Имя Отчество",
  "sex": "m" | "f",
  "birthYear": number | null,
  "deathYear": number | null,
  "status": "alive" | "deceased" | "unknown",
  "birthPlace": "string",
  "nationality": "string",
  "bio": "краткое описание",
  "photo": "filename.jpg" | null,
  "contacts": "string | null",        // только для живых
  "restPlace": "string | null",       // только для упокоенных
  "restMapUrl": "https://yandex.ru/maps/... | null",
  "parents": ["id", ...],
  "spouses": ["id", ...],
  "children": ["id", ...]
}
```

Нет фото → силуэт ♂/♀ по полю `sex`.

## Режим чтения (по умолчанию)

- Вертикальное дерево (предки сверху, потомки вниз).
- Карточки: стиль **Imperial Burgundy** — бордо-фон, золотая двойная рамка, орнаменты по углам, вензель, круглое фото в золоте, слабый золотой дамаск-узор.
- Поля в карточке: ФИО, годы (♔ род — упокоен), статус-бейдж, БИО (кратко, курсив), национальность, место рождения. Для упокоенных — ссылка «📍 на карте» (Яндекс.Карты). Для живых — контакты (если есть).
- Брак = сердечко между супругами. Золотые линии связей.
- Zoom/pan мышью. Клик по карточке → полная информация.

## Режим админа

- Ввод пароля → разблокирует редактирование. Пароль хранится в коде как base64 (слабая защита — осознанно принято, репа публичная).
- «+» на карточке → добавить родню, указать роль: муж / жена / сын / дочь / отец / мать.
- Правка любых полей карточки.
- Загрузка фото: сжатие/ресайз до ~400px в браузере перед сохранением.
- **Сохранение**: пользователь вводит персональный GitHub-токен → коммит `data.json` (+ новые фото) через GitHub REST API. Токен в коде НЕ хранится, спрашивается при каждом сохранении.

### Безопасность (осознанные ограничения)
- Сайт статический, репа публичная → `data.json` и пароль видны всем, кто откроет исходники. base64 = обфускация, не защита.
- Пароль отсекает только случайные правки гостей. Реальный контроль записи даёт GitHub-токен (его в коде нет).

## Калькулятор родства

- Выделить 2 карточки → показать «кто кому кем приходится».
- Алгоритм: граф персон → поиск ближайшего общего предка (LCA) → дистанции поколений с обеих сторон → термин.
  - Степень кузенства = min(дистанций) − 1; «removed» = разница дистанций.
  - Связи по браку (свойство) разрешаются через супругов.
- Поддержка: кровное родство + свойство (вариант 2): тесть, тёща, свёкор, свекровь, зять, невестка, сноха, сват, сватья, свояченица, свояк, деверь, золовка, шурин, ятровь и т.д.
- Сложный термин → раскрывается короткое пояснение + ссылка на источник (gramota.ru / Wikipedia «Свойство»).
- Таблицу терминов пишем сами по авторитетным источникам.

## Хостинг

- GitHub Pages, публичная репа. Фото в `/photos/` — лимитов с запасом хватает (десятки сжатых фото = единицы МБ).

## Не входит в объём (YAGNI)

- Серверный бэкенд, БД.
- Регистрация/многопользовательность.
- GEDCOM импорт/экспорт.
- Шифрование данных (репа публичная — бессмысленно).
